# UX-аудит Drivedocs

**Дата:** 2026-05-12
**Метод:** прогон по 27 критериям трёх принципов с позиции «новый пользователь, открыл первый раз»
**Сценарий:** Welcome → Wizard (6 шагов) → Home → Add Trip → Add Receipt → Reports → Settings
**Превью:** https://drivedocs-v2-9aa8d1qae-igorsibola.vercel.app

Удобство просело из-за жаргона, дизайн — из-за полузаконченной Visual Phase: 2 экрана перерисованы под warm-систему, 6 остаются на старой стилистике, что создаёт впечатление сборной солянки.

## Принцип 3 — МАКСИМАЛЬНАЯ ВЫГОДА (12/27 = 44%) ❌

| # | Критерий | Балл | Обоснование |
|---|---|---|---|
| 3.1 | Чёткое value proposition | **1/3** | Welcome: «Учёт служебного транспорта для ИП и ООО» — это категория, не выгода. «До 180 000 ₽ в год — не из кармана, из компании» — нумерический хук есть, но запрятан третьим буллетом. Нет фразы вида «Готовый путевой лист за 30 секунд вместо 30 минут в Excel». |
| 3.2 | Реальные результаты | **0/3** | Нигде нет social proof: ни количества пользователей, ни кейсов, ни отзывов, ни «12 000 листов выпущено». |
| 3.3 | Прозрачность цен | **1/3** | SettingsPage показывает «Pro / Бесплатный» и список фич Pro. Конкретной цены не видно (нет колонок «₽/мес»). На Welcome — ноль информации о платности. |
| 3.4 | Try-before-buy | 2/3 | Локалсторадж-режим даёт фактический бесплатный триал всего core-функционала без регистрации. Сильно. |
| 3.5 | Экономия времени | **1/3** | Автоматическая сборка путевого листа — это реальная экономия часов. Но **нигде в UI это не сформулировано** — пользователь должен сам догадаться. |
| 3.6 | Экономия денег | **1/3** | TaxBenefitBanner (₽25 800 за май, ~310 000 ₽/год в бизнес) **был удалён** с HomePage 2026-05-12 ради соответствия мокапу. Теперь экономия денег упомянута только на Welcome третьим буллетом, дальше — тишина. ReportsPage показывает только «N ₽ расходов», не «N ₽ налогового вычета». |
| 3.7 | Решение реальной боли | 2/3 | Путевые листы + чеки + годовой отчёт для ИП — это **топ-3 боли** этой ниши. Продукт реально лечит. Но месседжинг не подсвечивает «вот эту конкретную боль мы убили». |
| 3.8 | Долгосрочная ценность | 2/3 | Архив месяцев в Reports, накопительные отчёты, документы организации хранятся — да, ценность не убывает. |
| 3.9 | Честность | 2/3 | Free-тариф честно показывает что входит. Нет автосписаний/dark patterns в видимой части. Но в Welcome не сказано вообще ни слова про платность — это полу-обман. |

## 🎯 Топ-5 что чинить в первую очередь

### 1. Восстановить value-proposition выгоды на ключевых экранах (3.5, 3.6) — критично

**Не так:** TaxBenefitBanner снят с HomePage ради 1:1 соответствия мокапу. Welcome упоминает 180 000 ₽ третьим буллетом. ReportsPage показывает только сумму расходов, не сумму налоговой экономии. Главная выгода продукта — «деньги обратно через УСН» — не подсвечена нигде.

**Что делать:**
- Welcome: первый буллет → «Возвращайте до 180 000 ₽/год через налоговый вычет — без бухгалтера»
- ReportsPage hero-карта: добавить четвёртую метрику «Налоговая выгода: ~N ₽» (расчёт от tax-режима)
- HomePage: вернуть messaging выгоды, но **не как баннер из мокапа** — а как маленькую строку под KPI: «За май: 25 800 ₽ — расходы бизнеса, не из кармана». Решение «выкинуть» было ошибкой, потому что мокап рисовал штрафной alert как замену; реально нужно оба.

**Эффект:** ключевая бизнес-метрика «зачем платить за это приложение» становится видна. Подъём конверсии триала в платёж.

### 2. Прогнать оставшиеся 6 экранов под Warm-систему (1.2) — закрыть Visual Phase

**Не так:** HomePage + ReportsPage в новом стиле (Sora, oklch indigo, мягкие тени, 22px радиусы). TripsPage, SettingsPage, TripDetailSheet, QuickTripSheet, QuickReceiptSheet, OnboardingWizard — на старом синем Tailwind. Глобальный `MobileHeader` дублируется с page-headers новых экранов.

**Что делать:** идти по плану T-144 … T-149 — каждая страница 1 PR. SettingsPage особенно важен (плотно, дёрганая навигация, низкий contrast). При перерисовке скрывать `MobileHeader` так же, как уже сделано для `/home`.

**Эффект:** одно ощущение продукта, рост 1.2 / 1.8, доверие.

### 3. Контраст и доступность (1.7) — быстрая фикс-сессия

**Не так:** `text-slate-400` (#94a3b8) на белом — 3.3:1, проваливает WCAG 4.5:1 для нормального текста. Используется для KPI-лейблов, чек-категорий, sub-text карточек, дат — то есть везде. Focus-states не определены явно.

**Что делать:**
- Заменить `text-slate-400` → `text-slate-500` (#64748b = 7:1) для всего, что не Decorative
- Оставить `slate-400` только для иконок и декоративных глифов
- Добавить глобальный focus-ring через `:focus-visible` в `index.css`
- Чек contrast hero-карты Reports на градиенте (white text — нужен alpha-overlay для гарантии)

**Эффект:** доступность с 1 до 3, читаемость для возрастной аудитории (а это профильная для ИП).

### 4. Покрыть оставшиеся жаргонные точки HelpInfoSheet (2.3, 2.6)

**Не так:** F-021 закрыл EntityType, VehicleModel, Summary, Settings. **Не закрыты:**
- `TaxModeStep` — «УСН 6%», «УСН 15%», «Патент», «ЕСХН» без подсказки «какой выбрать если не уверен»
- `QuickReceiptSheet` — категории чеков без «что из этого налоговая принимает к вычету»
- `AddTripSheet` — поле «Цель поездки» без объяснения «зачем заполнять и что писать»
- `InnStep` — «ФНС» расшифровать или убрать

**Что делать:** добавить тот же паттерн `HelpInfoSheet` для четырёх точек. Создать `HELP_TAX_MODE`, `HELP_RECEIPT_CATEGORIES`, `HELP_TRIP_PURPOSE`, заменить «ФНС» на «налоговая».

**Эффект:** drop-off в onboarding на TaxModeStep и в первом чеке. Сейчас это, вероятно, самые «застревающие» точки.

### 5. Прозрачность цен на Welcome (3.3, 3.9)

**Не так:** Welcome молчит о платности — продукт выглядит «бесплатным», новичок ставит ожидание, потом в Settings обнаруживает Pro-тариф. Это лёгкий вариант обмана ожиданий.

**Что делать:** под кнопкой «Начать настройку» строкой `text-xs text-slate-500`: «Бесплатно до N поездок в месяц • Pro — XXX ₽/мес, без автосписания • Отмена в один тап». Если цены ещё не определены — хотя бы «Бесплатный план + платный позже».

**Эффект:** доверие. 3.3 поднимается с 1 до 2–3. Уменьшит negative bounce при первом столкновении с paywall.

## ✅ Что уже хорошо

- **Try-before-buy на localStorage** (3.4) — пользователь может полноценно покрутить продукт без регистрации. Редко кто так делает в РФ-сегменте — сильное преимущество.
- **F-021 help-layer** на 4 точках онбординга — это уровень, к которому большинство конкурентов не доходят. Расширить — и сильная фича.
- **Onboarding 6 шагов с back-навигацией и сохранением ввода** — методически правильно, без «опросника на 20 полей».
- **Reports hero-карта с готовым месячным экспортом в один тап** — закрывает главную боль ИП («чтобы было что показать налоговой»).
- **Wzard SummaryStep с кликабельными строками-«редактировать»** — UX-паттерн «увидел → сразу исправил», работает хорошо.
