Format: US-xxx · Роль · Цель · Value · Acceptance criteria · Feature/Task links
Status per story is derived from linked feature status.

## US-001 — Настройка workspace для ИП

**Как** предприниматель (ИП),
**я хочу** создать рабочее пространство и указать свой налоговый режим и правовую модель использования автомобиля,
**чтобы** приложение автоматически настроило нужный список документов и подсказки.

**Acceptance criteria:**
- Онбординг предлагает выбор типа предприятия ИП.
- После завершения wizard workspace отмечен как `isConfigured: true`.
- Список документов соответствует выбранной конфигурации (тип + налоговый режим + модель авто).
- Пользователь видит итоговый экран с кратким описанием своей схемы.

**Features:** F-001, F-002
**Tasks:** T-010, T-011

## US-002 — Настройка workspace для ООО

**Как** руководитель ООО,
**я хочу** создать рабочее пространство для своего юрлица и задать конфигурацию,
**чтобы** приложение учитывало специфику ООО (документы, налоговые требования).

**Acceptance criteria:**
- Онбординг предлагает выбор ООО.
- Конфигурация workspace хранит тип, налоговый режим, модель авто.
- Список документов отличается от ИП-конфигурации там, где есть отличия.

**Features:** F-001, F-002
**Tasks:** T-010, T-011

## US-003 — Переключение между предприятиями

**Как** пользователь с несколькими предприятиями,
**я хочу** быстро переключаться между workspace с телефона,
**чтобы** не путать данные разных предприятий.

**Acceptance criteria:**
- WorkspaceSwitcher открывается тапом на имя workspace в хедере.
- После переключения все данные (поездки, документы, события) обновляются для нового workspace.
- Активный workspace отмечен визуально (checkmark).
- Можно создать новый workspace прямо из switcher.

**Features:** F-001, F-015
**Tasks:** T-012

## US-004 — Быстрая запись поездки с телефона

**Как** пользователь,
**я хочу** за минуту зафиксировать поездку прямо с телефона,
**чтобы** не забывать записывать маршруты и вести корректный учёт.

**Acceptance criteria:**
- AddTripSheet доступен с любого экрана (кнопка + в хедере).
- Форма содержит: откуда, куда, расстояние (км), цель, дата (сегодня по умолчанию).
- После сохранения поездка появляется в TodayPage и TripsPage.
- На TodayPage появляется success banner "Поездка добавлена".
- Создаётся событие `trip_logged` в ленте.

**Features:** F-003, F-011
**Tasks:** T-020, T-024, T-045

## US-006 — Просмотр деталей поездки и удаление

**Как** пользователь,
**я хочу** открыть поездку и при необходимости удалить её,
**чтобы** исправлять ошибки в записях.

**Acceptance criteria:**
- Tap на TripCard открывает TripDetailSheet.
- Sheet показывает: полный маршрут (откуда/куда), расстояние, цель, дата.
- Кнопка "Удалить" требует подтверждения (inline confirm).
- После удаления sheet закрывается, поездка исчезает из всех списков.

**Features:** F-005
**Tasks:** T-051, T-053

## US-007 — Список необходимых документов

**Как** пользователь,
**я хочу** видеть список документов, которые нужны для моей схемы использования автомобиля,
**чтобы** понимать, что собрать и что уже готово.

**Acceptance criteria:**
- DocumentsPage показывает документы, сгруппированные по статусу.
- Прогресс-бар отражает долю готовых документов.
- Каждый документ имеет статус: Нужен / В работе / Готов / Просрочен.
- Пустое состояние если документов нет (workspace не настроен или нет шаблонов).

**Features:** F-006
**Tasks:** T-031

## US-008 — Детали документа и смена статуса

**Как** пользователь,
**я хочу** открыть документ и понять, зачем он нужен и как его подготовить,
**чтобы** действовать самостоятельно без обращения к бухгалтеру.

**Acceptance criteria:**
- Tap на документ открывает DocumentDetailSheet.
- Sheet содержит: статус, дедлайн (если есть), "Зачем нужен", "Как подготовить", tip (если есть).
- Кнопки: "Отметить как готовый", "Пометить В работе" (если не in_progress), "Отменить готовность" (если completed).
- Изменение статуса мгновенно отражается в DocumentsPage и HomePage AttentionSection.

**Features:** F-007
**Tasks:** T-032, T-033, T-052

## US-009 — Лента событий и напоминаний

**Как** пользователь,
**я хочу** видеть ленту всех событий (записанные поездки, напоминания, штрафы),
**чтобы** быть в курсе того, что происходит по моему предприятию.

**Acceptance criteria:**
- EventsPage показывает все события workspace в обратном хронологическом порядке.
- События разделены на "ленту событий" и "штрафы" (tabs или filter).
- Непрочитанные события визуально выделены.
- Tap на событие открывает EventDetailSheet и помечает его прочитанным.

**Features:** F-008, F-009
**Tasks:** T-041, T-042

## US-010 — Срочные документы и события на главном экране

**Как** пользователь,
**я хочу** видеть самое важное прямо на домашнем экране,
**чтобы** не пропустить срочные документы и события без захода в разделы.

**Acceptance criteria:**
- HomePage показывает секцию "Требуют внимания" при наличии срочных документов или событий.
- Документы со статусом required/overdue входят в секцию.
- События с severity urgent/warning входят в секцию.
- Показывается не более 3 элементов, остальные — по ссылке "Все →".
- Tap на документ → DocumentDetailSheet (без навигации).
- Tap на событие → EventsPage.

**Features:** F-010, F-012
**Tasks:** T-050, T-052

## US-011 — Управление рабочими пространствами

**Как** пользователь,
**я хочу** переименовывать workspace, добавлять новые и при необходимости сбрасывать конфигурацию,
**чтобы** поддерживать актуальные данные по каждому предприятию.

**Acceptance criteria:**
- SettingsPage позволяет переименовать текущий workspace.
- Можно добавить новый workspace (→ OnboardingWizard).
- "Сбросить конфигурацию" требует подтверждения и устанавливает `isConfigured: false`.
- После сброса при входе на HomePage показывается guard с CTA завершить настройку.

**Features:** F-001, F-013, F-015
**Tasks:** T-013

## US-QR01 — Быстрый ввод чека

**Как** пользователь,
**я хочу** быстро зафиксировать расход (чек) прямо с экрана "Сегодня",
**чтобы** не потерять информацию о топливе, парковке или ремонте, и иметь её под рукой для отчётности.

**Acceptance criteria:**
- Кнопка "Чек" на TodayPage активна и открывает QuickReceiptSheet.
- Sheet содержит: сумма (число), категория (топливо / парковка / ремонт / мойка / другое), дата (сегодня по умолчанию).
- Сохранение чека без суммы невозможно — кнопка неактивна.
- После сохранения sheet закрывается; чек записан в store.
- Закрытие без сохранения (backdrop или ×) не создаёт записи.

**Features:** F-QR01
**Tasks:** T-080, T-081, T-082

## US-AT01 — Релевантный список "Требуют внимания"

**Как** пользователь,
**я хочу** видеть в секции "Требуют внимания" только реально важные элементы — срочные документы и необработанные предупреждения,
**чтобы** не тонуть в шуме и сразу понимать, что требует действия.

**Acceptance criteria:**
- AttentionSection показывает только документы со статусом `required` или `overdue`.
- AttentionSection показывает только непрочитанные события с severity `urgent` или `warning`.
- Элементы отсортированы: `urgent` выше `warning`; документы выше событий при равном severity.
- Tap на документ → DocumentDetailSheet (без навигации).
- Tap на событие → переход на EventsPage.
- Если элементов нет — секция "Требуют внимания" не отображается совсем.
- Список формируется через `buildAttentionItems()` — одна точка правил.

**Features:** F-AT01, F-012
**Tasks:** T-083, T-084

**я хочу** видеть сохранённые чеки за сегодня прямо на экране "Сегодня",

**Features:** F-QR02
**Tasks:** T-085, T-086

## US-QR06 — Выбор периода для истории чеков

**Как** пользователь,
**я хочу** переключать период просмотра (7 / 30 / 90 дней) прямо на странице истории чеков,
**чтобы** не листать лишнее или, наоборот, видеть более долгую историю без лишних экранов.

**Acceptance criteria:**
- На ReceiptsPage сверху видны chips: «7 дней», «30 дней», «90 дней».
- Активный chip визуально выделен.
- По умолчанию выбрано «30 дней».
- Переключение обновляет список и аналитику без перезагрузки страницы.
- Если за выбранный период нет чеков — пустое состояние с подсказкой.
- Выбор периода — локальный state компонента, не сохраняется в store.

**Features:** F-QR03
**Tasks:** T-093

## US-AT02 — Предупреждение о непривязанных чеках

**Как** пользователь,
**я хочу** видеть напоминание на главном экране, если у меня есть чеки без привязки к поездке,
**чтобы** не забыть зафиксировать связь расхода с маршрутом до конца рабочей недели.

**Acceptance criteria:**
- Если за последние 7 дней есть хотя бы один чек без `tripId` — в AttentionSection появляется warning-карточка.
- Текст: «Есть чеки без поездки» + подзаголовок с числом и периодом.
- Tap на карточку → переход на ReceiptsPage (`/w/:id/receipts`).
- Если все чеки привязаны (или чеков за 7 дней нет) — карточка не показывается.
- Карточка не появляется, если ReceiptsPage открыта — это поведение уже обеспечивается тем, что attention items вычисляются на HomePage.

**Features:** F-AT02
**Tasks:** T-094, T-095

## US-QR04 — История чеков за период

**Как** пользователь,
**я хочу** видеть историю своих чеков за последние 30 дней,
**чтобы** контролировать расходы по машине, не ограничиваясь только сегодняшним днём.

**Acceptance criteria:**
- На TodayPage есть ссылка "Все чеки →", ведущая на ReceiptsPage.
- ReceiptsPage показывает все чеки за последние 30 дней, сортировка: новые сверху.
- Каждый чек показывает: сумму, категорию, дату, статус привязки к поездке.
- Если чеков за период нет — показывается понятный empty state.
- Tap на чек → ReceiptDetailSheet (можно привязать/отвязать поездку прямо из истории).

**Features:** F-QR03
**Tasks:** T-089, T-090, T-091

## US-QR05 — Аналитика расходов по категориям

**Как** пользователь,
**я хочу** видеть суммы расходов по категориям (топливо, парковка, ремонт, мойка, другое) за период,
**чтобы** понимать, куда уходят деньги на обслуживание автомобиля.

**Acceptance criteria:**
- На ReceiptsPage над списком чеков показывается блок аналитики.
- Блок содержит: итоговую сумму за период + разбивку по категориям (только те, где сумма > 0).
- Категории отсортированы по сумме (больше — выше).
- Аналитика строится только на чеках за выбранный период (те же 30 дней).
- Нет графиков/диаграмм — только числа.

**Features:** F-QR03
**Tasks:** T-089, T-090, T-092

## US-QR03 — Привязка чека к поездке

**Как** пользователь,
**я хочу** привязать чек к конкретной поездке,
**чтобы** расход был связан с правильным маршрутом и корректно учитывался при формировании отчёта.

**Acceptance criteria:**
- В ReceiptDetailSheet видно, к какой поездке привязан чек (или "Не привязан").
- Кнопка "Привязать к поездке" разворачивает список доступных поездок workspace внутри sheet.
- Пользователь выбирает поездку → `tripId` сохраняется в store → статус обновляется реактивно.
- Если чек уже привязан, есть действие "Отвязать" → `tripId` удаляется из записи.
- TripDetailSheet показывает количество привязанных чеков для данной поездки.

**Features:** F-QR02
**Tasks:** T-086, T-087, T-088

## US-016 — Ежемесячный отчёт о поездках

**Как** пользователь (ИП/ООО),
**я хочу** получить текстовую сводку поездок за текущий месяц и скопировать её,
**чтобы** передать данные в бухгалтерию без ручного перепечатывания.

**Acceptance criteria:**
- На TripsPage есть кнопка "Отчёт", открывающая MonthlyReportSheet.
- Sheet показывает: месяц, количество поездок, суммарный пробег, список маршрутов с датами.
- Список содержит только поездки текущего месяца.
- Кнопка "Скопировать отчёт" копирует форматированный plain-text через `navigator.clipboard.writeText()`.
- Если clipboard недоступен — показывается textarea с текстом для ручного копирования.
- При 0 поездках за месяц: список заменяется сообщением, кнопка копирования неактивна.
- После успешного копирования — кратковременный success feedback (иконка + "Скопировано").

**Features:** F-016
**Tasks:** T-097, T-098, T-099
