# Память

**Хранилище:**
- `MEMORY.md` — долгосрочная память (факты, решения, предпочтения)
- `memory/YYYY-MM-DD.md` — дневник дня
- `knowledge/` — база знаний (архитектура, баги, чеклисты)

## При старте сессии (обязательно)
1. Прочитай `MEMORY.md` и файлы `memory/` за последние 3 дня.
2. Если файла `memory/YYYY-MM-DD.md` нет — создай с датой, временем, темой.
3. Если папок `memory/` или `knowledge/` нет — создай.

## Во время работы (обязательно)
- Каждые ~5 сообщений — краткий итог в `memory/YYYY-MM-DD.md`.
- Новый факт → сразу в `MEMORY.md`.
- Правка/коррекция от пользователя → `memory/feedback_ТЕМА.md`.
- Один вопрос 2+ раза → ответ в `knowledge/` отдельным файлом.
- 10+ сообщений без записи — нарушение. Остановись и запиши.

## Управление контекстом
При заполнении контекста на 60–65% выполнить `/compact` с явным указанием что сохранить: активный task ID, последнее решение, текущий файл.

## Перед завершением сессии
- Итог в `memory/YYYY-MM-DD.md`: что сделали, что осталось, ключевые решения.
- Обнови `MEMORY.md` если узнал новые важные факты (макс. 200 строк).

## Формат дневника
```md
# YYYY-MM-DD

## Сессия N (HH:MM)
Тема: ...

### Сделано
- ...

### Решения
- ...

### TODO на потом
- ...
```
