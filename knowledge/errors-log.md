## 2026-05-15 · Wizard back arrow + auto-advance на radio-кнопках

**Симптом:** на любом шаге OnboardingWizard стрелка «назад» не работает. Тап по radio-опции (выбор ИП/ООО, налогового режима) сразу перебрасывает на следующий шаг, минуя проверку.

**Корневая причина:** в коммите `66d3b87` (фича Enter-submit) добавлена обёртка `<form onSubmit={handleSubmit}>`. По умолчанию любая `<button>` внутри `<form>` без явного `type="button"` ведёт себя как `type="submit"` — при клике сначала отрабатывает `onClick`, потом форма submit'ится. Эффект: стрелка назад дёргает `handleBack`, тут же `handleNext` отменяет переход.

**Фикс (коммит `c3054bb`):** убрана `<form>`, Enter-submit реализован через `onKeyDown` на `<div>`-обёртке, фильтрует по `target.tagName === 'INPUT' | 'TEXTAREA'`. Главная кнопка «Далее» получила `type="button"` + `onClick={handleNext}`.

**Урок на будущее:**
- Не оборачивать сложные UI в `<form>` ради одного Enter-submit. Всегда есть keydown-альтернатива.
- Если всё-таки `<form>` — каждая вложенная кнопка должна иметь `type="button"` явно, кроме одной submit-кнопки.
- При добавлении новой кнопки в место, где есть `<form>`, — добавлять `type="button"` сразу.

**Где могла быть та же проблема (проверить, если будут жалобы):**
- AddTripSheet — есть ли там `<form>`?
- QuickReceiptSheet — есть ли там `<form>`?
- OrgProfileSheet, DriversSheet, VehicleProfileSheet — есть ли `<form>`?

## 2026-05-13 · React error #185 (infinite render) в AddTripSheet

**Симптом:** при открытии «Создать поездку» приложение зависает / падает с React error #185.

**Корневая причина:** селектор `useWorkspaceStore((s) => s.trips.filter(...))` создаёт новый массив на каждом ре-рендере → ссылка меняется → компонент ре-рендерится → петля.

**Фикс:** использовать готовый мемоизированный селектор `useWorkspaceTrips(workspaceId)` из `workspaceStore`. Никогда не делать `.filter()/.map()` прямо в Zustand-селекторе.

**Урок:** в Zustand селектор должен возвращать **то же ссылочное значение** между рендерами, пока данные не поменялись. Любая трансформация (`.filter`, `.map`, `Object.values`) — это новая ссылка, нужен `useMemo` снаружи или специализированный мемоизированный селектор.
